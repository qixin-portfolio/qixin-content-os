## 齐鑫版新增规则

1. **V4 业务适配性**：只有能进入真实重复工作流的方法论才做成 Skill。
2. **隐私分级**：`public`、`internal`、`confidential` 决定可引用与可提交范围。
3. **公开仓库不保存源材料**：只保存来源清单、哈希和短引用。
4. **双宿主交付**：同时兼容 Hermes 与 Codex 的 `SKILL.md` 目录结构。
5. **默认 3–8 个 Skill**：避免知识资产膨胀和触发冲突。
6. **人工批准后安装**：生成目录与实际安装目录分离。
7. **不污染业务代码**：Skill 源文件集中在 `agent-skills/`，试点产物集中在 `knowledge-packs/`。
