# v0.1.0 Phase 1 Release Baseline

日期：2026-07-12  
阶段：Phase 1  真实事件卡 → 母内容 → 四平台内容草稿

## 完成目标

Phase 1 建立了从真实事件卡到母内容和四个平台内容草稿的最小可运行链路。

- 事件卡必须包含证据说明、结果和个人感受。
- 事实检查通过后，才能生成母内容草稿。
- 母内容可以适配为朋友圈、X、小红书和抖音草稿。
- 平台适配只改变表达方式，不改写事实。
- 保留人工审核节点，不自动调用外部模型，不自动发布。

## 数据模型

使用 Prisma + SQLite，包含以下实体：

- `Project`：项目及项目状态。
- `SourceItem`：原始素材，包含类型、内容、来源地址和可见性。
- `EventCard`：真实事件、问题、结果、个人感受和证据说明。
- `MasterContent`：标题、开头、故事、洞察、反思和行动号召。
- `PlatformVariant`：平台、标题、正文和草稿状态。
- `Asset`：事件卡关联的证据或媒体资产。
- `PublishRecord`：平台发布地址、时间和指标 JSON 字符串。

关系已配置：项目关联素材和事件卡，事件卡关联母内容和资产，母内容关联平台版本，平台版本关联发布记录。

## API

- `GET /api/events`：按创建时间倒序返回事件卡列表。
- `POST /api/events`：创建事件卡。
- 创建前调用 `factCheck`；缺少证据说明、结果或个人感受时返回 `400`，不会写入事件卡。

当前未提供母内容或平台草稿的持久化 API。

## 已知限制

- AI 内容生成使用本地 mock provider，尚未接入 OpenAI、Claude、DeepSeek 或豆包。
- 当前只实现事件卡列表和创建 API，母内容及平台草稿尚未落库和编辑。
- 测试覆盖领域逻辑，尚未覆盖真实 HTTP 请求和 SQLite 数据库集成。
- SQLite 为本地优先存储，尚未提供生产部署、备份和迁移流程。
- 不支持自动登录、自动发布或自动抓取外部平台数据。
