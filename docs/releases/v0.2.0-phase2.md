# v0.2.0 Phase 2 Release Baseline

日期：2026-07-12  
阶段：Phase 2  Reality Import Layer 真实项目资产导入层

## 完成目标

Content OS 已经可以接入真实项目资料，形成：

`Project → ProjectSource → SourceItem → EventCard`

本阶段只建立真实资料导入和事实转换边界，不做完整 CMS、自动发布或复杂后台。

## 数据模型

- 新增 `ProjectSource`：记录来源类型、名称、路径、仓库和 metadata JSON。
- `Project` 关联多个 `ProjectSource`。
- `ProjectSource` 可关联多个 `SourceItem`。
- `SourceItem` 增加 `sourceType`、`sourcePath` 和来源关系。
- `EventCard` 关联多个 `SourceItem`，保证事件卡有来源素材。
- 已新增 Prisma migration：`20260712110000_add_project_sources_and_traceability`。

## 项目与 API

- `GET /api/projects`：获取项目列表。
- `POST /api/projects`：创建项目。
- `POST /api/inbox/import`：按项目导入 `.md`/`.markdown` 文件，保存为 private SourceItem，并创建 ProjectSource。
- `GET/POST /api/events`：事件卡创建要求 `sourceItemIds`，并校验来源素材属于同一 Project。

页面：

- `/projects`：项目列表。
- `/inbox/import`：Markdown 导入页面。

## Import Service

- `src/lib/importers/markdown.ts`：纯函数导入 Markdown，保留原文并提取一级标题。
- `src/lib/importers/github.ts`：只导入指定 repository + commitSha，内容包含 commit message、changed files 和 commit URL；无法获取时返回明确错误。
- `src/lib/content/event-generator.ts`：mock provider 从结构化 SourceItem 生成 EventCard draft；缺少事实字段或来源 ID 时返回 validation error。

## 已知限制

- 本阶段只实现 Markdown 导入页面和指定 commit adapter，未做目录扫描、自动同步或 webhook。
- GitHub adapter 需要可访问 GitHub API；无 token 时只依赖公开 commit 权限。
- EventCard generator 使用 mock provider，只识别结构化 Markdown 事实段落，不接入外部模型。
- 本机 Prisma 7.8 的 `migrate dev`/`migrate deploy` 在 schema engine 阶段返回空错误；migration SQL 已由 `migrate diff` 生成，并用 `db execute` 验证可执行。
- seed 依赖本机真实资料路径；资料不存在时会失败，不会生成替代内容。
