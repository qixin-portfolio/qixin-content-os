## 重点行为审查

### 四个平台 revision 是否隔离

通过。

- `EditorialDraft` 使用 `@@unique([masterContentId, platform])`，四个平台各有独立草稿。
- 每个草稿有独立 `currentRevisionId`，字段本身带唯一约束。
- `DraftRevision` 使用 `editorialDraftId + revisionNumber` 唯一约束。
- 集成测试验证朋友圈新增 revision 后，X 草稿仍保持自己的 Revision 1。
- 透明工地 seed 实际创建：`wechat_moments`、`x`、`xiaohongshu`、`douyin` 四个独立 EditorialDraft。

### AI 建议是否会覆盖人工版本

自动覆盖：不会。

- 获取建议接口只读取并返回建议，不修改 EditorialDraft。
- 只有用户明确点击“采用建议”后，才调用 `createSuggestionRevision`。
- 采用建议创建新的 `ai_suggestion` revision，旧的 `human_edit` revision 永久保留。
- 原始 MasterContent 不会被修改。

因此更准确的边界是：AI 建议不会静默覆盖；人工明确采用后会成为新的当前 revision，但历史人工版本仍可追溯。

### 未批准时是否不会生成 VoiceSample

通过。

- seed 创建四个平台 EditorialDraft 时只创建 `ai_initial` revision 和 StyleReview。
- seed 不把 Phase 3 AI 草稿自动转为 VoiceSample。
- 当前本地数据库查询结果：4 个 EditorialDraft，VoiceSample 数量为 `0`。
- 只有人工提交真实文案，或人工批准 EditorialDraft，才会创建 VoiceSample。

### approved 后是否生成 VoiceSample

通过测试。

`tests/editorial/revision-service.test.ts` 验证：批准成功后会创建：

- `sourceType = approved_draft`
- `approved = true`
- `sourceReferenceId = EditorialDraft.id`

批准稿的标题、正文、平台和 VoiceProfile 会被保存为样本。没有人工批准，不会发生这一步。

### overallScore 低于 70 是否默认不能批准

通过。

- `approveEditorialDraft` 会在批准前重新执行 StyleReview。
- `overallScore < 70` 且没有 `overrideReason` 时抛出错误。
- 测试使用包含模板化开头、营销腔、绝对化表达和禁用词的低分稿验证了默认拒绝。

### override 是否必须填写原因

通过。

- 低于 70 分时，缺少 `overrideReason` 不能批准。
- 提供原因后才允许批准。
- override 原因会写入 `human_approval` revision 的 `changeSummary`。

### Hook 和 CTA 是否允许为空

通过。

- StyleReview 不会因为 Hook 或 CTA 为空而创建缺陷。
- 编辑页面明确标注“Hook（可以为空）”和“CTA（可以为空）”。
- 建议器返回 `allowEmptyHook = true`、`allowEmptyCta = true`。
- X 平台初始 EditorialDraft 的 CTA 可以为空。

### 为什么 overallScore 81 仍不能直接发布

评分计算为：

`(aiToneScore 100 + authenticityScore 35 + clarityScore 88 + (100 - salesToneScore 0)) / 4 = 80.75 ≈ 81`

81 只代表：按当前规则，文本通过了“进入人工批准流程”的最低质量门槛。它不代表：

- 已经像齐鑫本人；
- 已经完成事实人工复核；
- 已经得到齐鑫批准；
- 已经适合发布；
- 已经产生 VoiceSample。

`authenticityScore = 35` 的原因是当前没有任何真实、批准的 VoiceSample。系统只有 VoiceProfile 规则，没有足够的本人文案样本可比对，所以不能证明个人声音匹配。

本稿还保留报告腔、产品说明书腔和抽象 CTA。即使分数达到 81，仍必须经过齐鑫人工判断和编辑；系统没有自动发布路径。

## 已知限制

- StyleReview 是 deterministic rules，不是真实模型，无法理解复杂语境，也不能真正判断“像不像本人”。
- 当前没有真实批准 VoiceSample，`authenticityScore` 只能代表规则校准，不代表个人声音学习。
- `ai_suggestion` 采用是显式操作，历史版本保留，但采用后会成为新的当前 revision。
- 当前页面使用 textarea，不提供复杂富文本、评论协作或多人权限。
- 尚未接入自动发布、平台登录、RAG、向量数据库、多 Agent、多模型路由或发布数据回流。
- 本机 Prisma 7.8 的 `migrate dev/deploy` 仍存在 schema engine 空错误限制；本阶段 migration 已在临时 SQLite 用 `db execute` 验证。
- seed 有 Node.js `MODULE_TYPELESS_PACKAGE_JSON` 性能提示，不影响执行。
