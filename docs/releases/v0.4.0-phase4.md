# v0.4.0 Phase 4 Release Baseline

日期：2026-07-12
阶段：Phase 4 Voice Calibration & Editorial Workbench 个人声音校准与人工编辑工作台

## 阶段目标

把 Phase 3 的 MasterContent draft 放入人工编辑流程：

`MasterContent draft → StyleReview → 修改建议 → 人工/建议 revision → 人工批准 → VoiceSample`

本阶段不自动发布，不接平台账号，不接 RAG、向量数据库、多 Agent、多模型路由，不做全自动重写、团队协作或复杂富文本编辑器。

## 数据模型

- `EditorialDraft`：保存平台、当前标题、正文、Hook、CTA、状态、当前 revision、绑定 MasterContent 和 VoiceProfile。
- `DraftRevision`：保存 revision 编号、完整文本、变更来源、变更摘要；`editorialDraftId + revisionNumber` 唯一，旧版本永久保留。
- `VoiceSample`：保存平台、本人文案、来源类型、来源引用、质量评分、备注、批准和启用状态。
- `StyleReview`：保存总分、AI 腔、真实性、清晰度、营销腔评分及 JSON 问题/建议。
- 新增 enum：`EditorialStatus`、`ChangeSource`、`VoiceSampleSourceType`。
- `MasterContent` 关联多个 `EditorialDraft`；`VoiceProfile` 关联 `EditorialDraft` 和 `VoiceSample`。
- 新增 migration：`20260712130000_add_editorial_workbench`。
- migration 只新增 Phase 4 表和关系，不删除或覆盖已有 MasterContent、EventCard、SourceItem。

## StyleReview 规则

第一版使用 deterministic rules，不调用真实模型：

1. 模板化开头：`你有没有发现`、`在这个时代`、`随着 AI 的发展`、`今天想和大家分享`、`很多人都不知道`、`万万没想到`。
2. 课程广告腔：`赶紧收藏`、`建议点赞收藏`、`私信我领取`、`手把手教你`、`保姆级教程`、`闭眼冲`、`普通人也能逆袭`。
3. 过度总结：`这不仅仅是`、`这不是`、`归根结底`、`总而言之`、`让我们一起`等。
4. 虚假确定性：`一定`、`绝对`、`百分百`、`必然`、`彻底改变`。
5. 过度感叹号、标题强情绪标点和 Emoji 堆叠。
6. VoiceProfile 的 `avoidWords`。
7. 补充检测 Phase 3 已发现的泛化 Hook：`从……开始，记录……`。

评分含义：

- `aiToneScore` 越高，检测到的 AI/模板腔越少。
- `authenticityScore` 越高，越接近已有批准 VoiceSample；没有批准样本时只做规则校准，分数不会代表已学会个人声音。
- `clarityScore` 越高，结构越清楚。
- `salesToneScore` 越高，营销腔越重，越需要修改。
- `overallScore < 70` 默认不能批准。

## 版本管理规则

- 初始平台稿保存为 `ai_initial` Revision 1，保留原始 MasterContent 不动。
- 点击采用 AI 建议时创建 `ai_suggestion` 新 revision；建议接口本身不修改 EditorialDraft。
- 保存人工修改创建 `human_edit` 新 revision。
- 批准创建 `human_approval` revision，并保存批准时间。
- 所有 revision 永久保留；平台之间分别拥有自己的 `currentRevisionId`。
- 已批准稿再次修改时状态回到 `editing`。
- 批准前服务会重新运行 StyleReview；低于 70 分时没有 `overrideReason` 就拒绝批准。
- override 原因必须写入批准 revision 的 `changeSummary`。
- 拒绝原因写入拒绝 revision 的 `changeSummary`。

## VoiceSample 机制

- 用户通过 `/voice/samples` 手动提交真实本人文案，来源为 `manual_input`。
- 人工批准 EditorialDraft 后自动创建 `approved_draft` VoiceSample。
- Phase 3 AI 草稿不会自动变成 VoiceSample；seed 运行后样本数量仍为 0。
- 样本可设置质量评分 1-5、备注和启用/停用状态。
- 页面明确显示：当前只有规则，没有足够的本人文案样本，声音校准结果有限。
- 当前只能说“规则校准”，不能说“已经学会齐鑫语气”。

## 编辑工作台与 API

- `/editorial`：按待审核、编辑中、已批准、已拒绝展示 EditorialDraft、项目、EventCard、平台和 StyleReview 总分。
- `/editorial/[draftId]`：左侧事实来源/SourceItem/ContentAngle/VoiceProfile，中间 textarea 编辑区，右侧 StyleReview 问题和建议，底部 revision 历史及批准/拒绝。
- `/voice/samples`：手动添加、查看、评分和启停 VoiceSample。
- `GET/POST /api/editorial`：查看或按平台 VoiceProfile 准备四个平台草稿。
- `GET/PATCH /api/editorial/[draftId]`：查看详情或保存人工 revision。
- `GET/POST /api/editorial/[draftId]/suggestions`：读取建议或明确采用建议并新建 revision。
- `POST /api/editorial/[draftId]/review`：生成新的 StyleReview 历史记录。
- `POST /api/editorial/[draftId]/approve`：评分通过或人工 override 后批准并沉淀 VoiceSample。
- `POST /api/editorial/[draftId]/reject`：保存拒绝原因。
- `GET/POST /api/voice/samples` 与 `PATCH /api/voice/samples/[id]`：维护人工声音样本。

## 内容质量结论

事实保真通过，表达质量需要人工编辑。StyleReview 能识别 Phase 3 暴露的泛化 Hook，但真实性分数只有 35，原因是当前没有真实批准 VoiceSample。这个分数和规则建议不能证明已经学会齐鑫语气。

因此：Phase 4 可以作为工程稳定基线；不能作为可直接发布的内容基线。人工批准仍是必要节点。

## 已知限制

- deterministic StyleReview 不是语言模型，无法理解复杂语境或真正判断“像不像本人”。
- 没有真实批准 VoiceSample 时，authenticityScore 只能代表规则校准，不代表个人风格学习。
- 当前 suggestion 采用是显式操作，但仍以 `ai_suggestion` 记录；人工批准必须另行执行。
- 当前页面使用 textarea，不提供复杂富文本、评论协作或多人权限。
- 尚未接入自动发布、账号登录、RAG、向量数据库、多 Agent、多模型路由或发布数据回流。
- seed 运行会出现 Node.js `MODULE_TYPELESS_PACKAGE_JSON` 性能提示，不影响执行；本机 Prisma migration 工具限制仍存在。
