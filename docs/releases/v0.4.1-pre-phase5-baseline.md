## 1. d614080 VoiceSample 导入审查

### 能力检查

- 支持 `.csv` 和 `.json`；JSON 支持数组或 `{ samples: [] }`。
- `--dry-run` 只读取、校验和规划，不执行 `VoiceSample.create`；临时库数量保持不变。
- `body` 为空时拒绝。
- `platform` 只允许 `wechat_moments`、`x`、`xiaohongshu`、`douyin`。
- `qualityRating` 只允许 1 到 5 的整数。
- 未提供 `sourceType` 时默认 `imported_post`。
- 使用 `platform + SHA-256(trimmed body)` 去重。
- 既有 VoiceSample 进入去重集合，不覆盖原标题、正文、评分或来源。
- 同一导入文件内的重复记录会在第一条进入集合后被识别。
- 非法行单独进入失败列表，合法行仍可导入。
- 失败信息带 CSV 行号；JSON 使用记录所在输入序号对应的行号格式，不输出完整正文。
- 脚本不读取密钥、不调用网络、不输出完整私人正文到错误日志。
- 脚本只读取用户显式传入的文件，不复制文件、不自动 `git add` 文件。
- 脚本没有 PDF、OCR 或外部作者资料处理路径；本次没有将外部资料传给脚本。

### 临时 SQLite 验收

初始临时库包含 1 条合成的预存 VoiceSample 和四个平台的合成 VoiceProfile。表中的 `db_count` 是每个场景结束后的 VoiceSample 总数；失败数是审查器根据脚本输出的逐行失败原因统计的数量。

| 场景 | 成功数 | 跳过数 | 重复数 | 失败数 | db_count |
|---|---:|---:|---:|---:|---:|
| CSV dry-run | 2 | 0 | 0 | 0 | 1 |
| JSON dry-run | 2 | 0 | 0 | 0 | 1 |
| CSV 实际导入，含同文件重复和预存记录 | 2 | 2 | 2 | 0 | 3 |
| 重复导入同一 CSV | 0 | 4 | 4 | 0 | 3 |
| 非法 platform | 0 | 1 | 0 | 1 | 3 |
| 非法 qualityRating | 0 | 1 | 0 | 1 | 3 |

关键结果：

- 两次 dry-run 均显示成功候选，但数据库仍为 1 条。
- 实际导入后，预存记录仍保留原标题“原有标题”和评分 5，没有被输入文件中的新标题和评分覆盖。
- 同一正文在同一平台重复时跳过；同一正文在不同平台可以分别导入。
- 混合文件中的合法记录成功写入，非法 platform 只影响自身记录。
- 所有错误输出均为通用校验原因和行号，没有输出测试正文。

### 导入边界

当前脚本将实际导入的样本写为 `approved = true`、`active = true`。这意味着调用者必须先确认文件确实是齐鑫本人认可的真实内容；脚本本身不具备作者身份、版权归属或语义来源识别能力。未提供本人内容前，不执行实际导入。

`.gitignore` 已忽略 `storage/inbox/` 和本地数据库，但没有忽略所有任意路径的 `.csv`/`.json` 文件。实际使用应把导入文件放在 `storage/inbox/` 或仓库外，避免使用 `git add .` 时误提交私有文件。

## 5. 已知限制

1. 导入脚本不能判断输入正文是否属于齐鑫本人，也不能判断外部作者内容、转载内容或未经确认的内容。
2. 实际导入会标记样本为已批准和启用，因此必须由齐鑫在执行命令前完成来源确认。
3. 任意路径的 CSV/JSON 不会自动进入 Git，但仓库没有全局忽略所有 CSV/JSON；私有文件位置必须遵守约定。
4. 导入摘要输出失败原因和行号，但没有单独输出 `失败数量` 字段；审查时由逐行失败原因统计得到。
5. 当前去重只按平台和正文 hash，不判断标题、发布时间、图片或作者身份是否相同。
6. StyleReview 仍主要是 deterministic keyword rules，不能判断报告腔、语境、真实感受、幽默和“像不像齐鑫”。
7. 透明工地研究示例未经过齐鑫人工确认，不得直接发布或沉淀为 VoiceSample。
