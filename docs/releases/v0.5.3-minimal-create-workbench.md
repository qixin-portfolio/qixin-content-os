# v0.5.3 Minimal Content Creation Workbench

日期：2026-07-14  
状态：implementation 完成，待齐鑫人工确认  
分支：`codex/phase5.3-minimal-create-workbench`

## 为什么回到创作主流程

Phase 5.2 已具备事件、修订、声音样本和发布包等审计能力，但这些工程对象不是“今天写一条朋友圈”的自然入口。Phase 5.3 不再增加基础设施，而是把现有能力压缩为：

```text
输入一件事 → 选择选题 → 选择一版文案 → 人工修改 → 复制
```

`/create` 成为日常主入口；编辑、发布归档、选题库和素材来源保留在“更多”中。

## 用户主流程

1. 选择手动输入、最近项目或 X 收藏。
2. 手动写一句话，或选择一个有真实 EventCard 和 SourceItem 的最近项目。
3. 从三个用户可读选题中选择一个。
4. 在真实记录版、个人观点版、克制短版中选择一稿。
5. 在单一正文编辑器中人工修改。
6. 查看最多三条轻量风险和默认折叠的来源安全检查。
7. 复制纯正文；系统不自动发布。

透明工地资料整理已从最近项目默认列表排除，只能从“查看流程演示”主动加载，并明确标记为演示案例。

## localStorage 会话

版本化键：`qixin-content-os:create-session:v1`。

保存字段：`sourceMode`、`manualInput`、`selectedProject`、`topicCandidates`、`selectedTopic`、`draftCandidates`、`selectedDraft`、`editedContent`、`lightweightWarnings`、`assetSuggestions`、`currentStep`、`updatedAt`。

- 刷新和重新打开页面可恢复。
- 候选稿和人工正文分别保存，提示服务不覆盖人工内容。
- 损坏 JSON 或版本不兼容时安全回到新创作并显示说明。
- “清空本次创作”二次确认后只删除本机该键。
- 不与账号同步，不声称云端备份。

## 无持久化 API

### `POST /api/create/topics`

输入用户临时文本或项目安全摘要，返回三个朋友圈选题。每条只含标题、值得写的原因、推荐切入、平台和缺失信息。手动输入始终标记为待用户确认事实。

### `POST /api/create/drafts`

围绕同一来源和选题返回真实记录版、个人观点版、克制短版。API 只查询默认朋友圈 VoiceProfile 和 active/approved VoiceSample，没有 Prisma 写操作。

两条 API 均为 deterministic 本地 MVP，不声称调用真实 AI Provider。

## 声音样本边界

- 5 分样本权重高，4 分中高，3 分仅辅助。
- `approved_draft` 高于同质量 `imported_post`。
- 只参考句段结构、克制程度和现有 StyleReview 边界。
- 不复制 VoiceSample 原句，不学习错字、绝对化结论或连续感叹号。
- 当前 7 条样本全部只支持朋友圈，不声称已学会 X、小红书或抖音声音。

## 风险提示与复制

主界面最多显示三条自然语言提示，例如临时输入待确认、结论绝对、成果缺证据或项目报告腔。提示只提醒，不改正文。

“来源与安全检查”默认折叠，只显示来源摘要、未确认事实、隐私风险和配图注意。普通模式不显示 SourceItem、Revision、hash、评分或发布包信息。

“复制文案”只写入编辑器正文，不包含选题、标题标签、提示或来源。成功提示为“已复制，可以去朋友圈发布了。”，同时页面明确说明复制不会自动发布或写数据库。

## 已知限制

- 生成是 deterministic 本地 MVP，对任意生活输入的表达丰富度有限。
- 只支持朋友圈，暂不支持平台切换。
- 手动输入不是已验证事实，仍依赖齐鑫发布前确认。
- localStorage 不跨设备、不与账号同步，清除浏览器数据后无法恢复。
- 轻量提示以规则为主，不能替代人工语境判断。
- X 收藏研究库尚未接入当前版本，不读取 Phase 6A 私有资料。

## 后续判断
