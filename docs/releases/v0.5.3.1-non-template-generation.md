# v0.5.3.1 Non-Template Content Generation

日期：2026-07-14
分支：`codex/phase5.3-minimal-create-workbench`
基线：`7aae10f feat: build minimal content creation workbench`
状态：历史实现基线；后续结构化输出和 fallback 语义由 Phase 5.3.2 修正

## 1. 原问题

Phase 5.3 的选题和三稿由固定 TypeScript 字符串拼接。三个版本共同采用“输入复述 -> 抽象判断 -> 未完成或下一步”的结构，只调整顺序和长度。VoiceSample 只进入旧 StyleReview 的词语重合计算，没有影响正文结构，因此工程测试通过但真实文案验收失败。

完整模板审计见 `docs/research/create-template-generation-audit.md`。

## 2. 修正后的生成链路

```text
原始输入
  -> Provider.createBrief
  -> 服务层按原始输入收紧 ContentBrief
  -> Provider.createTopics
  -> 人工选择选题
  -> Provider.createDrafts
  -> 事实边界 + 结构相似度 + VoiceSample 整句复制检查
  -> 只重试重复版本一次
  -> 页面显示 generation mode 和质量状态
```

ContentBrief 保存用户明确提供的事件、细节、反应、矛盾、判断、未决问题、下一步、事实边界和外部观点归属。草稿 API 会再次按 `sourceText` 收紧浏览器回传的 Brief，不能通过篡改本地状态加入“客户认可”等新事实。

三稿的计划分别是：

- 真实记录版：先发生的事情，后已有感受或判断。
- 个人观点版：先用户已经表达的判断，再用输入中的事件支撑。
- 克制短版：2-4 个短段，只保留事件、一个已有判断和留白。

## 3. 火山方舟 Provider

- Provider ID：`volcengine_ark`。
- 服务端变量：`ARK_API_KEY`、`ARK_MODEL_ID`。
- `ARK_MODEL_ID` 只读取齐鑫从火山方舟控制台取得的真实模型 ID 或推理接入点 ID；代码不猜测或硬编码 `doubao-2.1`。
- API：`https://ark.cn-beijing.volces.com/api/v3/chat/completions`。
- 请求优先使用 `response_format: { type: "json_object" }`，响应继续经过 Zod。
- API Route 只调用统一 Provider factory，不创建 Ark 客户端。
- 测试使用 mock fetch，不调用真实 API。
- 本轮没有配置 Ark 参数，没有联网搜索、知识库或豆包助手调用。

本基线原行为是：缺少任一变量时直接使用 `deterministic_fallback`；Ark 请求、结构化输出或 Zod 校验失败时，整次操作安全重跑 fallback。页面固定显示：

> 当前使用本地演示生成，文案可能带有模板感。

因此本文件中的五组正文只证明页面链路、事实边界和相似度机制，不代表 Seed 2.1 的真实文案质量。

## 4. VoiceSample 实际使用

- 只读取 7 条 active、approved、`wechat_moments` 样本正文，不读取内部索引标题。
- `approved_draft` 高于同分 `imported_post`；5 分高权重、4 分中高、3 分辅助。
- 提取开头方式、观点位置、段落长短、留白、不确定性、自嘲和情绪结构。
- Ark prompt 只收到结构画像，不收到 VoiceSample 原句。
- 输出后检查是否复制任一 VoiceSample 完整句子。
- 不学习错字、绝对化表达、连续感叹号，也不声称已经完整学会齐鑫声音。

## 6. 相似度与失败语义

检查项包括：首句重复、连续相同句子、段落数量与长度结构、结尾结构、关键抽象判断、一个版本是否只是另一个的删短版，以及 VoiceSample 完整句复制。

只有命中的版本会调用 `regenerateDraft`，最多一次。再次失败时返回：

- `qualityStatus: insufficient`
- `qualityMessage: 三个版本仍然过于相似，请保留当前人工稿并稍后重试。`

候选重试只替换候选数组，不覆盖 localStorage 中已有的 `editedContent`。浏览器验收结果：输入 2、4 各重试一次，其余为 0；五条最终均为 `passed`。

## 9. 已知限制与下一步

- deterministic fallback 仍依赖规则拆句和排序，部分短稿机械、部分项目稿有复述，不是发布级质量。
- 当前相似度是结构启发式，不是语义模型，可能漏掉语义重复或误报短文本。
- VoiceSample 结构画像只基于 7 条朋友圈样本，不能证明完整学会齐鑫声音。
- Ark adapter 已通过 mock 测试，但 Seed 2.1 的真实结构化输出、自然度、超时和失败降级尚未实测。

下一步只允许齐鑫在本地配置 `ARK_API_KEY` 与真实 `ARK_MODEL_ID` 后，用必要的最少调用次数复跑五条验收。真实效果通过前，不冻结为发布级生成基线。
