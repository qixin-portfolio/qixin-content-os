所有请求使用 `doubao-seed-2-1-turbo-260628`、`json_object`、stream=false、严格模式、禁止 fallback。单项上限 60 秒，前一项失败即停止。

## 3. 正式架构调整

新链路：

```text
原始输入
  -> 本地 GroundingContext
  -> Ark 一次生成 3 个 topics
  -> 人工选择 1 个 topic
  -> Ark 一次生成 3 篇 drafts
  -> 本地事实、归属、相似度和模板检查
```

GroundingContext 只包含：

- rawInput
- sourceMode
- platform
- confirmedUserStatements
- externalOpinionMarkers
- prohibitedClaims
- missingContext

它不推断用户没有表达的情绪、结果、下一步、场景或成果。模型 ContentBrief 的 Schema、Provider 方法、API 字段和浏览器 localStorage 字段均已删除。显式本地演示内部仍可使用旧的本地拆句器，但不会进入 Ark 请求。

## 4. Topics API

`POST /api/create/topics`：

- 服务端从原始输入创建 GroundingContext。
- 只读高质量 VoiceSample，先生成聚合声音摘要；不传标题、正文或 ID。
- 初次只调用一次 `createTopics`。
- 模型输出仅包含正好 3 条 topics：title、focus、whyWorthWriting、angle、missingInformation、sourceGrounding。
- 服务端补充 generationMode、provider、model、Prompt 预算元数据和 lightweightWarnings。
- 不再返回模型生成的 ContentBrief。

## 5. Drafts API

`POST /api/create/drafts`：

- 不接受浏览器传入的 brief；服务端从 sourceText 重新建立 GroundingContext。
- 只发送原始输入、被选择的一条 TopicCandidate、GroundingContext 安全信息、精简声音摘要和三种叙事要求。
- 不重发其他两个 topics，不发送数据库内部对象或完整 VoiceSample。
- 初次一次返回 scene_record、thought_progression、restrained_short 三稿。
- 相似度或事实风险失败时只返回 insufficient，不自动发第二次模型请求。

## 7. 结构化输出与 fallback

- 继续使用 `json_object -> 安全 JSON.parse -> 轻量规范化 -> Zod`。
- 只允许移除完整单层 JSON Markdown 围栏。
- null/缺失数组可转为空数组，单字符串可转单元素数组。
- 不从长文本截取 JSON，不补事实、情绪、结论或下一步。
- 首次结构失败最多一次结构修复；第二次失败返回 `schema_validation_failed`。
- Provider timeout 从 120 秒降为 60 秒；Route 上限从 150 秒降为 75 秒。
- timeout、rate_limited、authentication_failed、schema_validation_failed、provider_error 不自动 fallback。
- 通用自动 fallback 辅助函数已删除。只有用户点击“使用本地演示生成”后，Route 才直接选择 LocalFallbackProvider。
