## 6. SourceItem 候选边界

候选仅保存：

- title
- relativePath
- sourceUrl
- author
- publishedAt
- modifiedAt
- tags
- contentHash
- 安全摘要
- riskFlags
- sourceCategory=`external_research`
- factEligibility=`unverified_reference`
- Obsidian 双链和附件的规范相对引用元数据

`SourceItem.content` 和 `SourceItemVersion.content` 是现有模型的必填字段，但 Phase 6A 写入值与安全摘要完全相同，不保存完整外部文章正文。临时库验证：两表 `content <> summary` 的记录均为 0，最大长度均为 181 个字符。

页面不以完整 `content` 作为摘要兜底；详情页只显示脱敏、限长的 `summary`。暂存层会再次限制摘要长度并脱敏，即使调用方伪造扫描结果也不能直接写入长正文或联系方式。

所有 SourceItem 和版本默认 `unverified_reference`；临时库 `confirmedFact` 记录为 0。代码中没有自动转为 confirmedFact 的路径。

### 隔离语义

- 上述 5 篇风险笔记默认不进入候选；其当前隔离状态写入人工维护的审计清单，不能因正文删掉风险片段而自动解除。
- 风险解除需要后续人工修改隔离清单；Phase 6A 没有自动解除入口。
- 页面和 CLI 只显示脱敏相对路径与风险类型，不输出电话、微信号、绝对路径或敏感正文。
- 新增高置信 API Key、Token、Cookie、JWT、云凭证检测；命中后标记 `secret_exposure`、立即 quarantine，并在摘要中替换为固定占位符。
- 如既有 SourceItem 后来变成风险、失去来源或删除，保留历史版本，但当前记录标记 `sourceMissingAt` 并更新风险类型；Topic 详情不再把它显示为当前有效证据。
- 图片附件本身不复制、不写数据库。
- 没有风险文件导入 API；暂存层仍会二次检查来源、路径、风险、固定分类和事实等级，不能只靠 `isSourceItemCandidate=true` 绕过。

## 8. 扫描、导入幂等与版本语义

临时 SQLite 和自动化测试确认：

1. 同一 Vault 重复扫描/暂存不新增 SourceItem。
2. 同一路径、同一 contentHash 不新增版本。
3. 同一路径内容变化创建新版本。
4. 旧版本永久保留，不覆盖历史快照。
5. 删除文件只设置 `sourceMissingAt`，不物理删除 SourceItem 或版本。
6. 文件恢复后清除 `sourceMissingAt`。
7. 文件移动后，旧相对路径标记 missing，新相对路径创建独立 SourceItem。
8. 同内容、不同相对路径创建两个独立 SourceItem 和独立版本。
9. 两个独立 PrismaClient 并发导入同一扫描时，唯一键 + 有界冲突重试确保最终只有 1 个 SourceItemVersion。
10. SourceItem、Version、ScanRun 在同一事务内；故障注入后全部为 0，不留半成品。
11. TopicCandidate 和关系也在同一事务内；关系插入故障时不留下半成品 TopicCandidate。
12. manifest 来源由 `a.md` 改为 `b.md` 后，仅删除旧的 `manifest reference`，保留人工关系边界。

重复暂存会新增一条 ScanRun 审计记录，这是预期行为；不会新增 SourceItem 或 Version。写入型 ScanRun 使用 `scanMode=staging_import`、`status=completed`；纯 CLI dry-run 不写数据库。

## 9. TopicCandidate 边界

- 私有 manifest 共 30 条，全部初始状态为 `proposed`。
- manifest schema 只接受 `proposed`，不能以 shortlisted/rejected/converted 作为初始状态。
- 证据强度分布：medium 18、weak 10、strong 2。两个 strong 是人工 manifest 明确填写，不是系统默认或自动推断；仍保持 `unverified_reference` 和版权/时效风险提示。

- 详情页对未补一手证据的选题显示“尚未填写”，并明确提示先补山西/交城本地一手资料再进入内容生产。

## 14. 版权、隐私与已知限制

### 版权与隐私

- 外部文章不批量保存正文，只保存安全摘要、hash、来源和相对引用。
- 对外使用必须保留来源 URL/作者，并采用最小必要引用；不得复制长段原文。
- 风险图片不复制、不 OCR；联系方式、路径、密钥和客户字段默认隔离/脱敏。
- 外部作者表达不进入 VoiceSample；外部案例不写成齐鑫实际经历。

### 已知限制

1. `/sources/obsidian` 在本地首次请求会同步扫描 Vault；当前 1153 文件规模可接受，但不适合公网并发。
2. staging 导入按记录执行多次 SQLite 操作；当前 164+30 规模可接受，扩大前应批量化。
3. Topic 列表尚未分页；扩大候选池前应增加 limit/分页和精简 select。
4. 当前是本地单用户应用，Topic POST 路由没有独立认证/CSRF 层；不得直接部署到不可信网络。
5. 高置信风险检测不是 DLP 完整替代；当前审计命中路径使用人工持续隔离清单，新风险仍需人工复核。
6. 每个选题都有通用的一手证据缺口提示，但 30 条尚未逐条填写专属 `firstHandEvidenceNeeded`。
7. SQLite 冲突重试同时依赖 Prisma code 与少量驱动错误文本；升级 Prisma/adapter 时需复测并发场景。
