# v0.6.0 Phase 6A：X 长文收藏研究库与选题候选池

日期：2026-07-13
基线：`4c83e25`；分支：`codex/phase6a-obsidian-research-source`

## 结论

本阶段完成了只读扫描、候选暂存和人工审核边界。Obsidian Vault 在 Content OS 中统一显示为“X 长文收藏研究库”，它是以 X 收藏长文为主、持续加入优质外部资料的动态研究库，分类仍为 `external_research`。本地目录名只用于定位来源，不能用于推断内容主题、事实真实性或证据强度。

外部资料默认 `factEligibility=unverified_reference`，只能作为选题线索和外部观点参考。没有自动进入 VoiceSample、EventCard、MasterContent 或发布流程。

## 数据模型

- `ProjectSource` 支持 `obsidian_vault`，保存 displayName、vaultKey、sourceCategory、rootFingerprint、lastScannedAt、enabled；Vault 绝对路径只从 `OBSIDIAN_RESEARCH_VAULT_PATH` 读取，不入库。
- `SourceItem` 增加 relativePath、sourceUrl、author、publishedAt、modifiedAt、tags、contentHash、summary、riskFlags、links、attachmentRefs、sourceMissingAt 等字段。
- `SourceItemVersion` 以 `sourceItemId + contentHash` 保存摘要版本；同一路径内容改变不覆盖历史，文件删除只标记 `sourceMissingAt`。
- `ScanRun` 保存临时 staging import 统计和 manifestHash；纯 CLI dry-run 不写数据库。
- `TopicCandidate` 保存受众、痛点、核心角度、证据强度、新鲜度、推荐平台、风险和审核状态；`TopicCandidateSource` 支持多来源与来源角色。

Migration：`prisma/migrations/20260714090000_add_phase6a_obsidian_research/migration.sql`。

## 扫描器边界

`src/lib/sources/obsidian/` 只读取 Markdown 和文件元数据，忽略 `.obsidian`、隐藏文件、临时文件和冲突副本。它提取 Frontmatter、标题、标签、来源 URL、作者、原发布日期、修改时间、双链、外链和附件引用；每篇计算 SHA-256 contentHash，所有输出均为 Vault 相对路径。

导入候选只保存安全摘要，不把完整外部文章批量写入业务表。附件只作为相对引用，不复制、不 OCR、不上传。扫描不会修改文件内容、文件时间或目录结构。

## 动态资料库语义

- Vault 会持续新增优质长文，扫描是可重复的增量发现过程，不是一次性导入。
- 每次扫描应区分 `discoveredNew`、`changed`、`unchanged`、`missing`、`quarantined`。Phase 6A 已具备 contentHash、版本保留、missing 和 quarantine 基础；完整的增量分类与提醒留待明确授权的后续阶段。
- 同一路径内容变化时创建新版本；原文件删除时保留历史记录，不物理删除 SourceItem 或旧版本。
- 单篇文章进入 Vault 不自动创建 TopicCandidate；用户人工筛选后才进入正式选题池。
- 来自 X 不等于真实，文章数据表现好也不自动提高 `evidenceStrength`。
- 主题和分类必须从笔记内容与人工选择产生，不从 Vault 文件夹名称推断。候选主题可包括 AI、GEO、SaaS、内容运营、自媒体、知识库、Agent、创业、产品设计、工作流和装修行业应用等。
- 外部作者表达不得进入 VoiceSample。

## 隐私、版权和事实边界

风险类型支持：`phone_number`、`wechat_contact`、`local_absolute_path`、`customer_privacy`、`unknown_source`、`copyright_risk`、`outdated_data`、`unverified_claim`。本次审计路径只保留相对路径；页面不展示电话、微信号或完整本地绝对路径，摘要会脱敏。

外部作者表达不进入 VoiceSample；外部案例不写成齐鑫完成的项目；外部收入、涨粉、客户、上线和行业预测不自动转为 confirmedFact；公开使用前仍需保留来源 URL、作者信息，并人工做最小必要引用。

## 页面与限制

- `/sources/obsidian`：Vault 名称、最近扫描、文件/Markdown、候选、重复/风险、来源完整度、断链和附件缺失。
- `/topics`：候选池、受众、痛点、角度、来源数、证据强度、新鲜度、平台、风险和状态。
- `/topics/[topicId]`：选题定义、来源摘要、风险、证据缺口、下一步调研动作和人工字段。
- 状态只支持 `proposed → shortlisted/rejected`、`rejected → proposed`。

已知限制：页面要么扫描当前环境变量指向的 Vault，要么读取已经暂存的数据库；manifest 预览状态不会直接写回真实数据库。Node dry-run 仍会提示模块类型性能 warning；Next build 会提示动态文件读取的 NFT trace warning，两者不影响结果。

## Phase 6B 建议

Phase 6B 应定义为“X 长文收藏库的增量同步与人工选题收件箱”，不定义为批量导入当前 164 篇某一行业资料。重点包括增量扫描、新增内容收件箱、内容变更提醒、人工 shortlist/reject、内容驱动主题聚类、选题去重和补一手证据提醒。

继续保持人工确认、版本保留、外部作者和事实边界；不因入库自动创建 TopicCandidate，不接自动 confirmedFact、自动 EventCard、自动成稿或自动发布。本阶段不实现 Phase 6B。
