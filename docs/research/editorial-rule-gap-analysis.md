# 编辑规则差异分析

## 分析范围

对比对象：

- `VoiceProfile`：平台语气、偏好词、禁用词和写作规则。
- `StyleReview`：Phase 4 的 deterministic rules 和评分。
- `rewrite-suggester`：可选建议、空 Hook/CTA 处理和人工采用边界。
- `docs/releases/v0.4.0-phase4-release-review.md`：透明工地朋友圈的实际审查结果。
- 本次外部写作方法研究及 `qixin-editorial-rules-candidates.md`。

本文件只做差异分析和研究示例，不修改代码、数据库或 VoiceSample。

## 1. 当前系统已经覆盖的规则

| 领域 | 当前覆盖 | 证据 |
|---|---|---|
| 事实边界 | EventCard 需要证据、结果和个人感受；生成前有 fact-check | Phase 1/3 事实检查与 SourceItem 追溯 |
| 来源追溯 | MasterContent 保存 SourceItem 引用 | Phase 2/3 数据模型和生成服务 |
| 平台隔离 | 四个平台有独立 EditorialDraft 和 revision | Phase 4 release review |
| 人工优先 | AI 建议只有明确采用才生成新 revision，原稿和人工版本保留 | `revision-service` 和 Phase 4 review |
| 低分审批 | `overallScore < 70` 默认不能批准，override 需要原因 | Phase 4 review |
| VoiceSample 门槛 | 未批准不生成 VoiceSample，批准后才沉淀样本 | Phase 4 review |
| 模板腔 | 检测若干模板开头和泛化 Hook | `template_opening`、`generic_generated_hook` |
| 营销腔 | 检测收藏、私信、教程、逆袭等固定表达 | `sales_tone` |
| 总结腔 | 检测若干过度总结和对仗短语 | `over_summary` |
| 确定性 | 检测“一定、绝对、百分百、必然”等表达 | `false_certainty` |
| 禁用词 | 检测 VoiceProfile `avoidWords` | `avoid_word` |
| 空字段 | Hook 和 CTA 可以为空，建议器不会强行补写 | `allowEmptyHook`、`allowEmptyCta` |
| 外部样本边界 | 当前批量导入脚本只处理用户提供的样本文件；本次 PDF 没有进入导入 | `scripts/import-voice-samples.ts` 和本次研究边界 |

## 2. 当前主要依赖关键词、但缺少语境判断的规则

### 2.1 具体事件和报告腔

StyleReview 能发现模板化 Hook，但不能判断一段文字是否真的从具体事件进入。透明工地稿删除 Hook 后 `overallScore` 从 78 到 81，仍然保留“产品一页纸、功能模块清单、行业案例说明”等资料归档式表达。系统能看见词，不能判断这些词在当前段落中是否有现场感。

### 2.2 事实先于抽象判断

系统能检查空字段和部分绝对化表达，但不能判断“产品已经形成”“项目有价值”等判断是否被当前段落中的具体证据支撑，也不能自动区分“产品文档已形成”和“产品已上线”。这需要事实引用和人工复核结合。

### 2.3 真实感受和第一人称

`rewrite-suggester` 在 VoiceProfile 要求第一人称、而正文没有“我”时会提示人工补充，但不会知道齐鑫是否真的有这个感受。自动补写第一人称会制造本人经历，风险高。

### 2.4 节奏、留白和结尾

当前规则可以发现模板句和总结词，却不能理解信息顺序是否自然，不能判断一段留白是克制还是遗漏，也不能判断 CTA 是否是一个真实的下一步。空 CTA 已被允许，但“应该为空”仍需要人工判断。

### 2.5 自嘲、幽默和反差

当前系统没有语境级的自嘲或幽默判断。它既不能判断自嘲是否降低姿态，也不能判断笑点是否改变事实；这是本次研究明确不建议自动化的部分。

### 2.6 authenticityScore 的含义有限

没有批准 VoiceSample 时，系统将真实性分数设为 35。存在批准样本后，当前计算仍主要依赖偏好词和词汇重合，不能等价于“已经学会齐鑫语气”。因此 Phase 4 的透明工地稿即使 `overallScore = 81`，也仍有报告腔、产品说明书腔和 AI 总结腔。

## 3. 后续可以实现的规则

以下仅是实现候选，不在本次任务修改：

1. 将每个正文段落与 SourceItem 或 EventCard 字段建立可见引用，人工可以逐段确认事实来源。
2. 对抽象名词密度、重复总结句、过长句子提供提示，但只给建议，不自动改写。
3. 对“已完成、上线、获得、增长、客户反馈”等状态词增加事实状态检查，要求对应证据或人工确认。
4. 增加“事实、感受、判断、计划”四类段落标注，帮助人工检查顺序和边界。
5. 记录人工对建议的接受、拒绝及原因，用于后续规则校准，而不是直接将拒绝文本当 VoiceSample。
6. 在有足够本人批准样本后，再评估结构特征、句长和词汇偏好；不以外部作者资料作为训练样本。

## 4. 暂时不应自动化的规则

- 判断一段话是否“像齐鑫”。
- 判断个人感受是否真实，或替齐鑫补写第一人称。
- 判断自嘲是否合适、是否会贬低自己或项目。
- 判断幽默、反差或包袱是否会改变事实。
- 判断一个结尾是否应该停住、升华或加入 CTA。
- 把“具体”自动扩写成场景、对白、动作和客户反应。
- 把外部作者的句式、节奏、口头禅或人格特征迁移到齐鑫内容。

## 5. 需要齐鑫人工判断的规则

- 哪些真实项目细节可以公开，哪些仍然私有。
- 哪个感受确实属于齐鑫，而不是模型根据文本推断出来的情绪。
- 哪些表达是齐鑫自然会说的话，哪些只是“看起来像内容”的句子。
- 哪些内容可以保留为未完成状态，哪些需要继续补证据后再写。
- Hook 和 CTA 是否为空，以及是否存在真实的下一步。
- 一篇稿件是否达到“可以审核发布”的个人标准。

## 6. 透明工地改写研究示例

### 6.1 边界

以下示例基于 Phase 4 release review 中的透明工地朋友圈草稿，只调整表达顺序和冗余，不增加用户数量、收入、成交、客户反馈、上线状态或测试结果。它只是研究示例，供齐鑫人工审核，不进入数据库，不进入 `VoiceSample`，也不代表齐鑫已经认可这版表达。

原稿已经确认的事实：

- 已形成产品一页纸、功能模块清单和行业案例说明等产品资料。
- 当前缺少截图、后台版本记录、代码路径和真实项目案例。
- 目前不能确认上线、客户、用户数量或收入。
- 个人判断是先保留证据缺口，不把产品规划写成已发生结果。

### 6.2 改写稿

标题：透明工地资料整理

Hook：空

正文：

> 透明工地的资料，先把已经有的和还缺的分开。
>
> 现在已有产品一页纸、功能模块清单和行业案例说明。继续核对时，截图、后台版本记录、代码路径和真实项目案例还没有补齐。
>
> 所以目前能确认的只有：产品文档已经形成。至于上线、客户、用户数量和收入，现有材料还不能证明。
>
> 先记录到这里。后面有新证据，再继续补。

CTA：空

### 6.3 修改标注

| 修改位置 | 使用的候选规则 | 说明 |
|---|---|---|
| Hook 留空 | `opening.concrete_event`、`cta.optional` | Phase 4 已发现原 Hook 是“从……开始，记录……”模板；没有必要补一个新的钩子。 |
| “已有”和“还缺的”并置 | `opening.concrete_event`、`evidence.before_abstraction` | 先给出项目材料状态，再进入判断，不添加新的项目经历。 |
| 保留产品资料清单 | `language.concrete_nouns` | 这些内容来自原稿和 SourceItem 记录，具体名词承担证据作用。 |
| 明确“还没有补齐” | `status.explicit_uncertainty` | 不能把证据缺口写成上线或成果。 |
| 保留“不能证明” | `humor.fact_preserving`、`ending.no_forced_elevation` | 不为了顺口删除事实限制，也不把计划写成完成。 |
| 结尾和 CTA 保持克制 | `cta.optional`、`ending.no_forced_elevation` | “后面有新证据，再继续补”仍是未来动作，没有写成已经补齐。齐鑫可以选择让 CTA 为空。 |

### 6.4 人工审查问题

这版仍需要齐鑫确认：

1. “先把已经有的和还缺的分开”是否准确表达本人实际判断。
2. “继续核对时”是否符合真实过程，若不准确应删除或改写。
3. “先记录到这里”是否是本人会使用的表达。
4. 是否需要保留任何第一人称；当前事实材料没有足够依据自动添加“我”。
5. 是否愿意公开产品资料名称和证据缺口。

## 结论

外部方法对 Content OS 最有价值的部分，是帮助人工发现具体事件、事实边界、冗余解释和强行升华之间的差异；它不能提供齐鑫个人声音。下一步若处理 VoiceSample，来源必须是齐鑫本人认可的真实文案，不能使用本次三份 PDF 或本次研究示例。
