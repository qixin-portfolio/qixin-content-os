# 齐鑫朋友圈 VoiceProfile Gap V1

## 审查对象

- 当前 VoiceProfile：`voice-wechat-default`，名称为“齐鑫朋友圈真实近况”。
- 当前语气：熟人感、克制、真实、带个人感受，不像课程广告。
- 当前偏好词：最近、折腾、没想到、慢慢发现、记录一下、还没完全做完。
- 当前禁用词：震撼、重磅、颠覆行业、普通人逆袭、年入百万、闭眼冲、赋能。
- 当前写作规则：第一人称、可承认失败和没做完、不超过500字、不堆Emoji、不强行引导私信、不使用成功学语气。
- StyleReview：模板开头、营销腔、过度总结、绝对化表达、情绪化标点、Emoji 和禁用词。
- rewrite-suggester：建议可选，Hook/CTA 可为空；正文没有“我”时可能提示补充第一人称。
- 外部候选规则：`docs/research/qixin-editorial-rules-candidates.md`，全部为 `proposed`。
- 真实样本：6 条 `wechat_moments`，评分为 5 分 1 条、4 分 3 条、3 分 2 条。

本文件不修改上述代码和数据，只分析差异。内部标题不是原始朋友圈标题，不能作为声音证据。

## 2. 当前规则错误限制的真实表达

### 2.1 “而是”不等于模板腔

StyleReview 将“不是……而是……”统一列为 `over_summary`。但 `cmriss3c00000ytupi7djcqg9` 中的观点改写和真实项目中的对比判断，可能需要这个结构。当前规则只能提示，不能据此自动删除。

### 2.2 没有“我”不等于不像齐鑫

rewrite-suggester 会在 VoiceProfile 要求第一人称、正文没有“我”时提示补充。5分样本 `cmriss3c60005ytup3tg6ss5a` 主要通过画面和感受成立，没有强行第一人称。不能为了满足规则虚构“我做了什么”。

### 2.3 强情绪词不必一律删除，但不能默认学习

真实样本确实出现强烈表达；StyleReview 可以提示，但应把它作为人工判断项。3分/4分样本中的强度不能直接升级为禁用词，也不能反向证明它是稳定声音。

### 2.4 句子长不等于 AI 腔

当前 StyleReview 主要看固定词和标点，没有判断中长句是否在自然推进思考。4分技术样本有较高概念密度，但不应因此直接被判为模板文案。

## 3. 当前规则缺失的特征

### 3.1 开头应按内容类型选择

真实样本同时存在画面开头、对象开头、引用开头和结论开头。系统没有区分生活观察和技术观点，不能只规定“从具体经历开始”或只规定“先给结论”。

### 3.2 技术事实到个人判断的转换

`cmriss3c50002ytupn7pzfhho`、`cmriss3c60003ytup84bxovhc`、`cmriss3c60004ytuphrf2d95n` 都把技术对象转成对创作、认知和工作方式的判断。当前规则只检查词，不检查这条转换路径。

### 3.3 生活内容的画面和留白

5分样本的力量来自等待、画面和不完全解释。当前系统没有画面密度、解释密度或结尾留白的判断能力。

### 3.4 质量加权

StyleReview 的 `calculateAuthenticity` 不使用 `qualityRating`，所有 approved VoiceSample 等权参与匹配。3分样本中的宣传感、Emoji 和强结论可能与5分样本产生同等词汇信号。

### 3.5 内部标题污染真实性匹配

当前导入标题是 Content OS 内部索引，不是原始朋友圈标题；但 `calculateAuthenticity` 会把 `sample.title + sample.body` 一起拆词。内部标题可能影响真实性分数，且不代表本人原文表达。

### 3.6 原始表达的结构信号

当前规则不判断段落数量、段落间空行、观察/例子/判断的顺序、引用后改写或结尾是否停在画面。

## 4. 不应自动化判断的部分

- 一句话是否真的像齐鑫，而不是只匹配词汇。
- 某个强烈判断是当时真实情绪，还是不应复用的偶发夸张。
- 自嘲是否降低姿态，还是在贬低本人、项目或事实。
- 生活内容是否应该留白，技术内容是否需要先给结论。
- “而是”结构是否自然，不能用固定关键词替代语境判断。
- 是否应该加入第一人称；没有真实依据时不能补写。
- 当前稿是否达到齐鑫本人愿意发布的标准。

## 5. 可进入后续代码实现的候选规则

本节只提出候选，不在本阶段实现：

1. `authenticity.quality_weight`：StyleReview 的真实性参考按 `qualityRating` 加权，5分高于4分，4分高于3分。
2. `authenticity.exclude_internal_title`：真实性匹配只使用原始正文，不使用 Content OS 内部标题。
3. `structure.content_mode`：将生活观察、社交记录、技术观点分为不同人工 benchmark 模式，避免一套开头规则覆盖全部内容。
4. `structure.observation_to_judgment`：提示人工检查是否存在对象/事实、个人解释和判断之间的连接，不自动生成缺失段落。
5. `structure.open_ending`：只提示结尾是否有强行升华或 CTA，不把省略号自动视为声音特征。
6. `language.contextual_contrast`：把“不是……而是……”从固定过度总结规则改为低置信度提示，保留人工决定。
7. `voice.provenance`：样本增加原始标题与内部索引标题的明确分离，避免导入标题污染声音学习。
8. `benchmark.human_label`：保存齐鑫对“像本人、事实准确、愿意发布、需修改”的人工标签，但不自动转成 VoiceSample。

## 结论

6条真实样本已经足够暴露当前 VoiceProfile 的方向正确处和评分盲点，但不足以自动定义完整声音。当前最重要的差异不是增加更多禁用词，而是让系统区分质量权重、内部标题、内容类型和事实到个人判断的结构。下一步仍应保持人工选择和人工修改。
